## 已知风险与待办

- GitHub Actions `29194195041` 在 Docker Hub 登录步骤失败；需要更新仓库 Secret `DOCKER_TOKEN`，否则节点丢失后无法从仓库重拉本版本。
- 分钟和并发限制仍是进程内实现；两个副本下理论上约为 12 次/分钟、2 个并发。MySQL 每日预算仍是全局原子限制。
- IP 维度会让同一 NAT 后的用户共享部分额度，这是当前产品取舍。
- Session 签名从硬编码值切到安全派生值，已有登录 Cookie 会要求用户重新登录一次。
- 全局请求体默认上限从 512 MiB 降为 32 MiB；旧 multipart 或大 ZIP 请求超过该值会收到 413。
- `/app/files` 仍是 Pod 本地临时存储，双副本不共享且重建会丢失。后续应迁移到 COS 或持久化共享卷。
